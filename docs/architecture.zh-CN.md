# 架构

[English](architecture.md) | **简体中文**

AnyaDance 由三个运行时部分组成：

1. `anyadance_core`：无外部依赖的 C++17 逻辑，由驱动、测试界面与测试共享。
2. `driver_anyadance.dll`：由 SteamVR 加载的 SteamVR/OpenVR 服务端驱动。
3. `AnyaDance.exe`：基于 Dear ImGui 的 Win32/DX11 测试发送端。

## 核心库

核心库拥有数据结构与可测试的行为：

- 设备常量与公共标识符
- 向量/四元数辅助函数，线缆姿态使用 XYZW 顺序的四元数
- UDP 协议解析与序列化
- `Y <= 2.0 m` 安全钳制
- 规范化 T 姿势重置
- 键盘输入映射（每个按键直接映射到一个按住的按钮或轴）
- 鼠标操控运算
- 日志环形缓冲区与操控合并

## 原生驱动

SteamVR 驱动最多注册六个设备：

- 头显（HMD）
- 左、右 `knuckles` 控制器
- 髋部、左脚、右脚通用追踪器

驱动在 `127.0.0.1:39570` 上启动一个回环 UDP 接收器。有效的样本会更新各设备的姿态状态，无效数据包将被忽略。作为纵深防御，驱动在数据包校验后会再次钳制设备 Y。

所有设备都以中性姿态开始且保持有效，即使数据包停止也仍然有效。驱动将最近接受的姿态报告为已连接、有效且 `TrackingResult_Running_OK`。

## 测试工具

伴随工具有一个 UI 线程和一个推流线程。UI 线程负责 ImGui 渲染、聚焦时的键盘轮询以及鼠标操控。推流线程复制同步后的状态，每帧序列化一次，并以 60 Hz 发送完整的六设备帧。UI 日志显示会改变状态的发送；未变化的保活数据包则保持安静。
