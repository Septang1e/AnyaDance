## T 姿势常量

伴随 UI 会以一个规范的测试 T 姿势启动并重置。这些相对头显的本地位置偏移被有意集中管理，且便于在测试中调整：

```text
头显 Y：           1.50 m
左控制器：         (-0.62, -0.17, -0.20) m
右控制器：         ( 0.62, -0.17,  0.20) m
髋部：             ( 0.00, -0.43, -0.05) m
左脚：             (-0.09, -1.24,  0.10) m
右脚：             ( 0.09, -1.24,  0.10) m
```

当头显处于中性位置 `(0.00, 1.50, 0.00)` 时，双脚位于 `(-0.09, 0.26, 0.10)` 与 `(0.09, 0.26, 0.10)`。

重置仅保留头显的 X/Z 与头显偏航——它会摆正头部，去除俯仰与横滚，使整个身体朝向一致的方向。其他设备位置由这些相对头显偏航的偏移构建。其他设备旋转仅使用头显偏航，再加上规范的本地控制器旋转：

```text
左控制器：  (0.0, 0.0, -0.7071067811865475, 0.7071067811865475)
右控制器：  (0.0, 0.0,  0.7071067811865475, 0.7071067811865475)
```

髋部与双脚使用与头显偏航复合的单位本地旋转。

## 虚拟头显显示

虚拟头显默认报告一个 `1920x1080` 的桌面窗口，以及每只眼睛 `64x64` 的推荐渲染目标，以尽量降低合成器负载。默认的 `left` 眼模式以单只眼睛填满窗口；`both` 模式将窗口拆分为两半并保持眼睛宽高比。SteamVR 设置可以在不重新构建驱动的情况下覆盖桌面窗口与每眼渲染目标，相关项位于 `driver_anyadance` 设置小节中的 `headset_window_width`、`headset_window_height`、`headset_render_width`、`headset_render_height`、`headset_window_eye_mode` 与 `headset_window_preserve_aspect`。提高渲染目标会显著增加合成器负载。

## UI 对称

伴随 UI 在控制器方框之间有一个对称复选框，在脚部方框之间也有一个。启用后，拖拽任意一侧会使另一侧成为绝对的镜像姿态。在 HMD 参考系模式下，镜像使用由当前头显偏航与头显位置定义的本地 YZ 平面。在 Global 参考系模式下，则以头显位置为中心、使用世界轴。这能保持成对设备的对称，而不仅仅是复制或取反拖拽增量。

捕获面板保留一块固定高度的鼠标帮助区域，使切换语言时设备行保持稳定。方框会随面板高度缩放。鼠标左键拖拽以本地 X/Y 移动非头显设备，中键拖拽旋转，右键拖拽移动深度。头显可旋转，并可通过左键+右键拖拽进行垂直（Y）移动，受 `kMaxDeviceY` 限制。

在面板空白区域拖拽用于驱动右控制器摇杆：按下点为中性中心，拖拽偏移映射到摇杆轴，钳制在 ±1，松开后回到中位。该功能用于操作右手快捷菜单（按住 `M`）。

## UI 日志

无论选择何种界面语言，伴随 UI 都将 UDP 日志保持为英文。悬停日志行的任意部分会显示发送的 JSON。点击某行会高亮它并打开一个固定的详情窗口，其中带有可滚动的 JSON 视图；再次点击高亮行会关闭详情窗口。仅当焦点变化释放了活动的控制器输入时，才会记录焦点变化。

## 本地化

UI 字符串位于 `src/ui/localization.*`。逐行存放字符串的表按代码与显示名称拥有可用的语言定义；界面渲染会遍历该表，而 UDP 日志标签则有意使用英文字符串。
