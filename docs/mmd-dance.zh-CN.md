# MMD 舞蹈

[English](mmd-dance.md) | **简体中文**

伴随 UI 可以把 MMD 舞蹈（一个 `.vmd` 动作）在内存中实时播放到六个虚拟设备上。Blender + MMD Tools 会基于真实模型完成精确的 FK/IK 解算；UI 再做一个小的重映射到硬编码骨架，并像其他姿态一样以 60 Hz 通过 UDP 推送。

它可通过 UI 主控制区右栏的 **舞蹈 (MMD)** 按钮进入，该按钮会打开一个小对话框：选择一个 VMD 和一个模型，点击 **分析**，然后 **播放**。

## 你需要准备

- **Blender**（自动从 `C:\Program Files\Blender Foundation\Blender *`、`ANYADANCE_BLENDER` 环境变量或 `PATH` 检测）。
- **MMD Tools**（[MMD-Blender/blender_mmd_tools](https://github.com/MMD-Blender/blender_mmd_tools)），作为 Blender 插件/扩展安装（自动从 `%APPDATA%` 下的 Blender 扩展目录树检测）。
- 一个 **VMD** 动作文件。
- 动作所针对的 **PMX/PMD 模型**。MMD 模型属于第三方作品并带有各自的授权，因此需由你自行提供。选择该舞蹈所对应的模型效果最佳。

## 工作原理

```text
.vmd + .pmx
  -> Blender + MMD Tools 导入并求值 FK/IK/约束          (scripts/blender_export_mmd.py)
  -> 一份世界空间关节姿态的已解算动作 JSON              (src/core/solved_motion.*)
  -> 简化重映射到六个硬编码设备                          (src/core/mmd_retarget.*)
  -> 60 Hz UDP 推送到驱动                                (UI 的常规路径)
```

`scripts/blender_export_mmd.py` 在 Blender 内无界面运行。它导入模型与动作，让 MMD Tools 求值姿态，并写出 **一个** JSON 文件（每次导出复用/覆盖，位于 `%TEMP%\AnyaDance\mmd_solved.json`），按帧保存头部、肩、肘、腕、骨盆、踝与脚趾的世界空间姿态，以及每只手的手指弯曲。输出采用与驱动相同的约定：右手坐标系，`+Y` 向上，`-Z` 向前，单位为米，四元数为 `xyzw`，虚拟形象的左侧在 `-X`。

## 简化重映射

由于骨架是硬编码的、转身由摇杆完成，重映射因此十分精简（`src/core/mmd_retarget.cpp`）：

- **转身 → 摇杆。** 重映射将世界朝向的转向保留为控制器输入。你用与常规姿态推流相同的 `Q`/`E` 控制在世界中转动虚拟形象。
- **按身高缩放。** 身体被缩放，使其站立高度匹配目标身高（`1.5 m`，即骨架的头显高度 `kResetHmdY`）。
- **直接放置设备。** 每个设备跟随其驱动关节：头显 ← 头部（上移到头顶），髋部 ← 骨盆，双脚 ← 踝，双手 ← 腕。手部以手掌为目标（腕沿前臂延伸），并围绕肩部略微拉伸（**手部伸展**），以便给 VRChat IK 一个手臂伸展的目标。
- **头显/髋部/双脚的旋转作为增量。** 朝向是关节 *相对于模型静止姿态* 的旋转，应用到干净的直立/朝前设备静止姿态上，使这些设备保持稳定的静止参考系。
- **控制器朝向来自前臂。** 已解算的腕四元数是模型骨骼的参考系。控制器的中性食指轴对齐到 肘→腕 前臂，绕其的横滚来自腕旋转，经由在静止姿态标定的腕本地扭转轴。
- **手指。** 当模型带有手指骨时，每只手的弯曲驱动控制器的骨骼手部姿态。

锚定：当你按下播放时，舞蹈会被钉到头显当前的 X/Z 位置，使虚拟形象原地起舞。

## 对话框

| 字段       | 含义                                                         |
|------------|--------------------------------------------------------------|
| VMD 动作   | `.vmd` 舞蹈文件。                                            |
| 模型       | 用于解算的 `.pmx`/`.pmd` 模型。                              |
| 循环       | 舞蹈结束时重复播放（否则保持最后一帧姿态）。                 |

其余一切（目标身高 `1.5 m`、播放速度、手部伸展、解算帧率）均使用固定默认值。

**高级**（可折叠）包含 **Blender 路径** 与 **MMD Tools 路径**。对话框打开时它们会用检测到的位置自动填充，因此大多数用户保持检测到的值即可。手动设置的路径会保存在 UI 偏好中，并在下次启动时复用。

**分析** 运行 Blender 解算并报告时长 / 帧数 / 缩放。**播放** 在解算成功前（以及解算进行期间）保持禁用，随后播放已解算的舞蹈。**停止** 返回 T 姿势。**重置为 T 姿势** 也会停止播放。

## 保存与加载片段（.nya）

舞蹈分析完成后，**保存 .nya** 会把重映射后的结果写入 `.nya` 片段文件。**加载 .nya** 读取后即可立即启用播放——加载片段会跳过 Blender 解算与重映射，因此已保存的舞蹈在后续运行（以及未安装 Blender 的机器上）可瞬间播放。

`.nya` 文件存储设备级帧（六个设备姿态加上每只手的手指弯曲），因此它与主窗口 **保存姿势** / **加载姿势** 所用的格式相同：姿势就是单帧片段。加载时设备 Y 钳制到 2 m 上限，手指弯曲钳制到 `[0, 1]`。参见 `src/core/nya_format.*`。

## 注意与限制

- 长舞蹈的首次解算可能需要 Blender 数秒到一分钟；其间 UI 会持续渲染并推送 T 姿势。
- 质量取决于动作与其目标模型的匹配程度。差异极大的模型（比例非常不同）仍能播放，但可能看起来不对。
- 这是离线解算 + 实时播放：已解算的动作由 UI 用于播放。项目[免责声明](../DISCLAIMER.md)中关于 VRChat 反作弊的常规注意事项同样适用。
